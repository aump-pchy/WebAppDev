🟣 Railway

สร้าง Project บน Railway

เชื่อม GitHub repo กับ Railway เพื่อ deploy backend

1

เข้า railway.app

คลิก Login → เลือก Login with GitHub → อนุญาต Railway ให้ access GitHub

2

New Project

คลิกปุ่ม New Project → เลือก Deploy from GitHub repo

3

เลือก repo

ค้นหา teaching-log → คลิก Deploy Now

4

ตั้ง Root Directory

ไปที่ Settings → Source → ตั้ง Root Directory เป็น backend

💡 Railway จะ auto-detect ว่าเป็น Node.js project และรัน npm start ให้อัตโนมัติ
🟣 Railway

ตั้งค่า Environment Variables

ไปที่ Variables tab แล้วเพิ่มทีละตัว

⚙️ Variables ที่ต้องใส่
Variableค่า
SUPABASE_URLhttps://xxxx.supabase.co (ขอจาก bro)จำเป็น
SUPABASE_SERVICE_ROLE_KEYeyJhbGci... (ขอจาก bro)จำเป็น
JWT_SECRETตั้งเองได้ เช่น teaching2569secretจำเป็น
FRONTEND_URLhttps://teaching-log.vercel.app (ใส่ทีหลัง)ทีหลัง
PORT3000optional
⚠️ FRONTEND_URL ใส่ทีหลังได้ — ต้อง deploy Vercel ก่อนถึงจะรู้ URL หรือถ้ายังไม่รู้ใส่ * ไปก่อนได้
📋 เช็คก่อนไปต่อ
ใส่ SUPABASE_URL แล้ว
ใส่ SUPABASE_SERVICE_ROLE_KEY แล้ว
ใส่ JWT_SECRET แล้ว
🟣 Railway

Deploy Backend และ copy URL

Railway จะ build และ deploy อัตโนมัติ

1

รอ Build เสร็จ

ดูที่ Deployments tab — รอจนขึ้น ✅ Success

2

Generate Domain

ไปที่ Settings → Networking → Generate Domain → Railway จะสร้าง URL ให้เช่น teaching-log-backend.up.railway.app

3

ทดสอบ API

เปิด browser ไปที่ URL นั้น ควรเห็น {"message":"Teaching Log API 🟢"}

4

Copy URL เก็บไว้

เอา URL นี้ไปใส่ใน Vercel ขั้นตอนถัดไป รูปแบบคือ https://xxx.up.railway.app

💡 ถ้า build fail ดู log ที่ Deployments → View Logs แล้วแจ้ง bro ได้เลย
⚫ Vercel

สร้าง Project บน Vercel

เชื่อม GitHub repo กับ Vercel เพื่อ deploy frontend

1

เข้า vercel.com

คลิก Sign Up → เลือก Continue with GitHub

2

Add New Project

คลิก Add New → Project → ค้นหา teaching-log → คลิก Import

3

ตั้ง Root Directory

ที่ Configure Project → ตั้ง Root Directory เป็น frontend

4

Framework Preset

Vercel จะ auto-detect เป็น Vite อัตโนมัติ ไม่ต้องแก้อะไร

⚫ Vercel

ตั้งค่า Environment Variables

ใส่ก่อนกด Deploy — ที่ Environment Variables section

⚙️ Variables ที่ต้องใส่
Variableค่า
VITE_API_URLhttps://xxx.up.railway.app/api (URL จาก Step 4)จำเป็น
⚠️ ต้องขึ้นต้นด้วย VITE_ เสมอ ไม่งั้น Vue จะมองไม่เห็น variable นี้
💡 ตัวอย่าง
VITE_API_URL = https://teaching-log-backend.up.railway.app/api
✅ ต้องมี /api ต่อท้าย URL ด้วยนะ เพราะ frontend เรียก endpoint แบบ /api/logs, /api/auth/login
⚫ Vercel

Deploy Frontend

กด Deploy แล้วรอ Vercel build เสร็จ

1

กด Deploy

คลิกปุ่ม Deploy — Vercel จะ run npm run build และ deploy อัตโนมัติ

2

รอ Build เสร็จ

ใช้เวลาประมาณ 1-2 นาที รอจนขึ้น 🎉 Congratulations!

3

Copy Vercel URL

Vercel จะให้ URL เช่น teaching-log.vercel.app — copy ไว้

4

อัปเดต Railway FRONTEND_URL

กลับไปที่ Railway → Variables → แก้ FRONTEND_URL เป็น URL ของ Vercel เพื่อให้ CORS ทำงานถูกต้อง

💡 ทุกครั้งที่ push code ขึ้น main branch — Vercel และ Railway จะ auto-deploy ให้อัตโนมัติเลย
✅ เสร็จสิ้น

เช็ค JWT_SECRET ใน Railway ว่าใส่ครบหรือเปล่า
