01

ทำไมต้องมี Vue?

HTML ปกติทำอะไรไม่ได้บ้าง

🤔 ปัญหาของ HTML ล้วนๆ
❌ HTML ล้วนๆ — ข้อมูลแข็ง
<!-- เปลี่ยนชื่อต้องแก้ไฟล์ --> <h1>สวัสดี สมชาย</h1> <p>อายุ 20 ปี</p> <p>แผนก IT</p>
❌ข้อมูลฝังอยู่ใน HTML ต้องแก้ทีละจุด ถ้ามี 100 หน้าต้องแก้ 100 ครั้ง
✅ Vue — ข้อมูลอยู่ที่เดียว
<!-- เปลี่ยนแค่ที่ ref() ที่เดียว --> const name = ref('สมชาย') const age = ref(20) <h1>สวัสดี {{ name }}</h1> <p>อายุ {{ age }} ปี</p>
✅ข้อมูลอยู่ที่เดียว แสดงได้หลายจุด แก้ครั้งเดียว อัปเดตทุกที่
🔄 Vue เชื่อม Data กับ UI อัตโนมัติ — ลองพิมพ์
ชื่อ (ref)
หน้าเว็บที่แสดงผล
สวัสดี สมชาย!
Vue อัปเดตทันทีที่ข้อมูลเปลี่ยน — ไม่ต้อง refresh
💡นี่คือ Reactivity — Vue คอยดูการเปลี่ยนแปลงของ data แล้วอัปเดต UI ให้เองอัตโนมัติ

ไฟล์ .vue คืออะไร?

3 ส่วนในไฟล์เดียว — template, script, style

📁 Single File Component (SFC)
🟩 <template>
HTML ที่แสดงบนหน้าจอ — ใส่ {{ }} และ v-* directive ได้
🟦 <script setup>
Logic ทั้งหมด — ประกาศ ref(), function, computed ที่นี่
🟪 <style scoped>
CSS เฉพาะ component นี้ — ไม่กระทบ component อื่น
<template> <div class="card"> <h1>{{ name }}</h1> </div> </template> <script setup> import { ref } from 'vue' const name = ref('สมชาย') </script> <style scoped> .card { padding: 20px; } </style>

ref() + {{ }} — Variable ของ Vue

นี่คือ JS concept แรกที่จะเจอ — Variable ที่ Vue มองเห็น

⚡ JS Concept — Variable คืออะไร
Variable = กล่องเก็บข้อมูล · ตั้งชื่อกล่องได้ · เรียกใช้ได้ทีหลัง

// JS ปกติ — let/const let name = 'สมชาย' // เปลี่ยนได้ทีหลัง const school = 'วท.เลย' // เปลี่ยนไม่ได้แล้ว
ใน Vue จะใช้ ref() แทน เพราะ Vue ต้องการ "มองเห็น" การเปลี่ยนแปลง
import { ref } from 'vue' const name = ref('สมชาย') // Vue มองเห็น ✓
📦 ref() — ประกาศ + ใช้งาน
ใน <script setup>
import { ref } from 'vue' // ประกาศ ref const name = ref('สมชาย') const age = ref(20) const dept = ref('IT') const active = ref(true) // ↑ string, number, boolean ได้หมด
ใน <template>
<!-- ใช้ {{ }} เพื่ออ่านค่า --> <h1>{{ name }}</h1> <p>อายุ {{ age }} ปี</p> <p>แผนก {{ dept }}</p> <!-- คำนวณใน {{ }} ได้เลย --> <p>ปีเกิด {{ 2568 - age }}</p>
⚠️ใน script ต้องเขียน .value เช่น name.value = 'ใหม่' แต่ใน template ไม่ต้อง Vue จัดการให้

v-bind — เชื่อม ref กับ Attribute

{{ }} ใช้กับข้อความ · v-bind ใช้กับ attribute ของ HTML tag

🔗 v-bind คืออะไร?
❌ HTML ปกติ — class แข็ง
<div class="bg-blue-500"> class เป็น blue-500 ตลอด </div>
✅ v-bind — class จาก ref
<div :class="bgColor"> class เปลี่ยนตาม ref </div> // script setup const bgColor = ref('bg-blue-500')
💡:class คือ shorthand ของ v-bind:class — ใช้ : นำหน้า attribute ที่ต้องการ bind
🎨 v-bind attribute ที่ใช้บ่อย
<!-- :class — bind Tailwind class --> <div :class="cardColor">...</div> <!-- :src — bind รูปภาพ --> <img :src="profileImg" :alt="name"> <!-- :href — bind ลิงก์ --> <a :href="profileUrl">ดูโปรไฟล์</a> <!-- :style — bind CSS inline --> <div :style="{ background: brandColor }">...</div>
🎮 ลองเปลี่ยนสี v-bind Interactive
เลือกธีมสี
🎨 ธีม: สีฟ้า
Tailwind class ที่ bind
<div :class="'bg-blue-500 text-white'">
⚡ JS Concept — Template Literal
ต้องการ class หลายอัน หรือเชื่อมข้อความกับ variable ใช้ template literal
const color = ref('blue') // Template Literal — ใช้ backtick ` และ ${} const bgClass = ref(`bg-${color.value}-500`) // ↑ แทรก variable เข้าไปได้เลย
